تُعد الـ **Enums** (الاختصار لـ Enumerations) من الأدوات القوية التي تجعل الكود الخاص بك أكثر نضجاً وتنظيماً، وهي مهارة أساسية لأي مطور برمجيات يسعى لكتابة "Clean Code".

---

## 1. ما هو الـ Enum؟
الـ Enum هو نوع بيانات مخصص (User-defined data type) يسمح لك بإعطاء **أسماء نصية** لمجموعة من **الأرقام الصحيحة**. بدلاً من التعامل مع أرقام مجردة لا معنى لها (مثل 1، 2، 3)، تتعامل مع كلمات واضحة (مثل Red, Blue, Green).

### كيف تفيدك كمطور (Developer)؟
* **قراءة الكود (Readability):** قراءة `if (color == Red)` أسهل بكثير من `if (color == 1)`. الرقم 1 قد يعني أي شيء، لكن كلمة "Red" واضحة.
* **منع الأخطاء (Type Safety):** الـ Enum يحصر الاختيارات في نطاق محدد، مما يقلل من احتمالية إدخال قيم خاطئة.
* **سهولة الصيانة (Maintainability):** إذا أردت تغيير قيمة "Red" من 1 إلى 10، تغيرها في مكان واحد فقط (تعريف الـ Enum) وسيتحدث الكود كله تلقائياً.
* **التوثيق الذاتي (Self-Documenting):** الكود يشرح نفسه بنفسه دون الحاجة لكثرة التعليقات.

---

## 2. شرح الكود بالتفصيل

هذا الكود يهدف إلى تغيير لون خلفية شاشة الـ Terminal (الكونسول) بناءً على اختيار المستخدم.

### الجزء الأول: تعريف الـ Enum
```cpp
enum enScreenColor { Red = 1, Blue = 2, Green = 3, Yellow = 4 };
```
* هنا قمنا بإنشاء نوع جديد اسمه `enScreenColor`.
* ربطنا الاسم `Red` بالقيمة الرقمية `1` وهكذا. الـ Enum في الخلفية يخزن أرقاماً، لكنه يظهر لك كأسماء.

### الجزء الثاني: واجهة المستخدم ومدخلات البيانات
```cpp
int color;             // متغير لتخزين الرقم الذي سيدخله المستخدم
enScreenColor Color;   // متغير من نوع Enum (كأنك تقول: هذا المتغير لا يقبل إلا ألوان الشاشة)

cin >> color;          // استقبال الرقم من المستخدم
```

### الجزء الثالث: عملية الـ Casting (التحويل)
```cpp
Color = (enScreenColor) color;
```
* **لماذا التحويل؟** المستخدم قام بإدخال قيمة من نوع `int` (رقم صحيح)، بينما المتغير `Color` من نوع `enScreenColor`.
* لأن C++ لغة صارمة في أنواع البيانات، لا يمكنك وضع `int` داخل `Enum` مباشرة، لذا نستخدم `(enScreenColor)` لإخبار المترجم: "حول هذا الرقم إلى النوع الخاص بالألوان".

### الجزء الرابع: التحقق وتغيير اللون
```cpp
if(color == enScreenColor::Red) // المقارنة باستخدام الرقم مباشرة
{
    system("color 4F"); // أمر نظام لتغيير اللون (الخلفية حمراء والكلام أبيض)
}
else if ( Color == enScreenColor::Green) // المقارنة باستخدام متغير الـ Enum بعد التحويل
{
    system("color 2F"); // (الخلفية خضراء)
}
```

* **ملاحظة تقنية:** الكود الخاص بك استخدم الطريقتين للمقارنة:
    1.  قارن `color` (الـ int) بـ `enScreenColor::Red`.
    2.  قارن `Color` (الـ Enum) بـ `enScreenColor::Green`.
* كلاهما سيعمل، لكن الطريقة الثانية (استخدام متغير الـ Enum) هي الأكثر احترافية لأنها تعتمد على النوع الذي عرفته خصيصاً لهذه المهمة.

---

## نصيحة إضافية لك كمطور:
بما أنك مهتم بأسلوب الـ Clean Code، يفضل دائماً عند استخدام الـ Enums مع خيارات متعددة استخدام جملة **`switch`** بدلاً من `if-else` المتكررة، لأنها مصممة خصيصاً للتعامل مع الحالات (Cases) وتجعل الكود أنظف وأسرع في القراءة.

مثال سريع:
```cpp
switch (Color) {
    case enScreenColor::Red:
        system("color 4F");
        break;
    case enScreenColor::Blue:
        system("color 3F");
        break;
}
```

عندما نتحدث عن كيفية عمل الـ **Enum** في الذاكرة، فنحن نتحدث عن البساطة والكفاءة المتناهية. الـ Enum ليس "كائناً" معقداً، بل هو في الحقيقة مجرد غطاء للأرقام الصحيحة.

إليك ما يحدث خلف الكواليس في الذاكرة عند تنفيذ كودك:

### 1. تخزين تعريف الـ Enum (وقت التجميع - Compile Time)
عندما تكتب:
```cpp
enum enScreenColor { Red = 1, Blue = 2, Green = 3, Yellow = 4 };
```
* **لا يحجز مساحة في الذاكرة (RAM)** في هذه المرحلة. 
* المترجم (Compiler) يقوم فقط بإنشاء "جدول رموز" (Symbol Table). هذا الجدول يقول للمترجم: "أينما وجدت كلمة `Red` في الكود، استبدلها بالرقم `1`". 
* هذا يعني أن الـ Enum **لا يستهلك مساحة** أثناء تشغيل البرنامج كتعريف، بل هو وسيلة لتنظيم الكود فقط.

---

### 2. المتغيرات في الذاكرة (وقت التشغيل - Runtime)
عندما تصل لمرحلة التنفيذ داخل الـ `main`:
```cpp
int color;           // يحجز 4 bytes (حسب نوع المعالج)
enScreenColor Color; // يحجز 4 bytes أيضاً!
```

* **متغير `Color`:** على الرغم من أن نوعه `enScreenColor` إلا أن المعالج يراه كـ `Integer`. في معظم الأنظمة الحديثة، يتم حجز مساحة **4 Bytes** لمتغير الـ Enum، وهي نفس المساحة التي يحجزها الـ `int`.
* **القيمة:** عندما تقوم بعملية الـ Casting وتخزين القيمة:
    `Color = (enScreenColor) 1;`
    الذاكرة لا تخزن كلمة "Red"، بل تخزن الرقم `1` فقط كقيمة ثنائية (Binary).

---

### 3. عملية الـ Casting (التحويل) في الذاكرة
عندما كتبت:
```cpp
Color = (enScreenColor) color;
```
* في الذاكرة، ما حدث هو مجرد **نقل (Move)** للقيمة من عنوان الذاكرة الخاص بـ `color` إلى عنوان الذاكرة الخاص بـ `Color`.
* التحويل هنا هو "تحويل منطقي" للمترجم ليتوقف عن إظهار تحذيرات (Warnings)، لكن بالنسبة للـ CPU، هو ينقل رقماً من مكان لآخر فقط.

---

### ملخص شكل الذاكرة (Memory Map) لنفرض أن المدخل كان `3`:

| اسم المتغير | العنوان (افتراضي) | القيمة المخزنة (Hex/Dec) | النوع الحقيقي للمعالج |
| :--- | :--- | :--- | :--- |
| `color` | `0x7ffe...1` | `3` | `int` |
| `Color` | `0x7ffe...5` | `3` | `int` (لكنه Enum للمبرمج) |

---

### لماذا هذا فعال جداً؟
1.  **السرعة:** التعامل مع الأرقام (Integers) هو أسرع شيء يمكن للمعالج القيام به.
2.  **المساحة:** الـ Enum لا يضيف أي ثقل (Overhead) على البرنامج؛ كود يستخدم Enum وكود يستخدم أرقام عادية سيكونان متطابقين تماماً في حجم ملف الـ `exe` النهائي وسرعة التنفيذ.

**باختصار:** الـ Enum هو "خدعة" بصرية مريحة للمبرمج، لكنها تتحول لأرقام بسيطة وسريعة جداً داخل الذاكرة.


بناءً على طلبك، سنقوم بكتابة برنامج يطلب من المستخدم اختيار دولة من قائمة، وبناءً على اختياره سنعرض له "العاصمة" الخاصة بها. سنستخدم الـ **Enum** لجعل الكود منظماً واحترافياً.

### البرنامج بلغة ++C

```cpp
#include <iostream>
using namespace std;

// تعريف Enum للدول
// أعطينا كل دولة رقماً تعريفياً يبدأ من 1
enum enCountry { Palestine = 1, SaudiArabia = 2, Egypt = 3, Yemen = 4 };

int main() {
    cout << "****************************\n";
    cout << "Select a country to see its capital:\n";
    cout << "(1) Palestine\n";
    cout << "(2) Saudi Arabia\n";
    cout << "(3) Egypt\n";
    cout << "(4) Yemen\n";
    cout << "****************************\n";
    cout << "Your Choice: ";

    int choice;
    cin >> choice;

    // السطر الذي يحتاج فهم: عملية التحويل (Casting)
    enCountry selectedCountry = (enCountry)choice;

    cout << "\nThe Capital is: ";

    // استخدام الـ Enum داخل جملة switch بدلاً من if
    switch (selectedCountry) {
        case enCountry::Palestine:
            cout << "Jerusalem (Al-Quds)";
            break;
        case enCountry::SaudiArabia:
            cout << "Riyadh";
            break;
        case enCountry::Egypt:
            cout << "Cairo";
            break;
        case enCountry::Yemen:
            cout << "Sanaa";
            break;
        default:
            cout << "Unknown choice!";
            break;
    }

    cout << "\n****************************\n";

    return 0;
}
```

---

### شرح السطر الذي يحتاج فهم

السطر الأهم والمحوري في هذا الكود هو:
```cpp`
enCountry selectedCountry = (enCountry)choice;
````

**لماذا هذا السطر وماذا يفعل؟**

1.  **اختلاف الأنواع:** المتغير `choice` هو من نوع `int` (رقم صحيح عادي)، بينما المتغير `selectedCountry` هو من نوع `enCountry` (الذي عرفناه نحن بالأعلى).
2.  **التحويل القسري (Casting):** الـ Compiler في ++C لا يسمح لك بوضع قيمة `int` مباشرة داخل متغير من نوع `Enum` بدون تأكيد منك، لأن الـ Enum يعتبر "نوعاً خاصاً".
3.  **وظيفة `(enCountry)`:** هذه تسمى **Explicit Casting**. أنت تخبر المترجم: *"أنا أعلم أن هذا الرقم هو مجرد `int` ولكني أريدك أن تعامله الآن كواحد من القيم الموجودة في قائمة الدول (enCountry)"*.
4.  **في الذاكرة:** هذا السطر لا يغير القيمة، فإذا أدخل المستخدم الرقم `1` في المتغير `choice` سيتم نقل الرقم `1` نفسه إلى `selectedCountry` ولكن بصفته "ممثلاً" عن القيمة `Palestine`.

**لماذا استخدمنا هذه الطريقة؟**
لأنك بمجرد تحويل الرقم إلى `Enum` (أي تخزينه في `selectedCountry`) أصبح بإمكانك كتابة `case enCountry::Palestine` في الـ `switch` بدلاً من كتابة أرقام مجردة مثل `case 1`. هذا هو جوهر كتابة كود نظيف (Clean Code) يفهمه البشر بسهولة.
